### 1) 해상도 가이드 (디펜스 게임에서 무난한 실무 기준)

- **권장 원본 제작 크기:** 타워 1종당 **`256x256` PNG (투명 배경)**  
  - 이유: 현재 가변 `tileSize`에서 축소 렌더링해도 디테일이 유지되고, 관리가 단순함.
- **경량 대안:** `128x128` (아주 단순 도형/아이콘 스타일일 때만)
- **권장 화면 점유율:** 실제 렌더링은 타일의 **70~78%**
  - 현재 코드 기준(`tileSize * 0.7`)과도 맞아 바로 적용 가능
- **앵커 기준:** 오브젝트 중심 정렬(중앙 앵커), 그림자/글로우는 PNG 내부 여백으로 처리
- **파일 포맷:** `PNG-24 + alpha` (웹P보다 RN/Expo에서 초기 운용이 편함)

즉, 이번 1차는 **`256x256 PNG 정지 일러스트 3장`**이 가장 안전하고 일반적인 선택이다.

## 이전 방식 vs 현재 방식

### 1) 발사 시작점 피드백
- **이전:** 타겟 위치에만 단일 원형 이펙트 생성  
- **현재:** 타워 위치에서 먼저 **recoil(발사 반동) 링+코어 플래시** 표시 후, 타겟 임팩트 표시

### 2) 공격 타입별 시각 차이
- **이전:** `single/slow/aoe` 모두 거의 같은 원형 효과  
- **현재:**
  - `impact_single`: 좁고 날카로운 타격(세로로 압축된 임팩트)
  - `impact_slow`: 링 + 내부 청색 코어(감속 느낌)
  - `impact_aoe`: 큰 외곽 링 + 내부 확산형 코어
  - `item_aoe`: 광역 폭발 전용 스타일

### 3) 코드 구조
- **이전:** `AttackEffect`는 색/크기만으로 렌더  
- **현재:** `effectType` 기반 렌더 분기 추가로, 같은 데이터 구조에서 표현력 확장
