주제: React Native 기반 '경로 고정형' 디펜스 게임 개발 기획

1. 개발 환경
도구: Cursor Pro (Vibe Coding 방식)
기술 스택: React Native + Expo (하이브리드 앱)
그래픽/사운드: AI 생성 에셋 활용 예정

2. 게임 컨셉 (A형식: 경로 고정형)
정해진 경로를 따라 적이 침공하고, 지정된 위치에 타워를 건설하는 방식.
난이도 시스템: 초보(자원 풍부, 적 약함), 중수(표준), 고수(자원 부족, 적의 특수 능력) 모드 포함.

3. 1단계 확정안 (코어 플레이 루프 구축)
목표: "웨이브 시작 -> 적 이동 -> 타워 자동 공격 -> 처치/누수 -> 승패 판정"까지 1판 완주 가능 상태를 만든다.

3.1 범위
- 맵: 고정 경로 1개, 타워 설치 가능 타일 8~12개
- 방어 캐릭터(타워): 3종 (상세는 `Character.md`)
- 적 캐릭터: 2종 (상세는 `Character.md`)
- 웨이브: 총 3개
- UI: HUD, 웨이브 시작 버튼, 일시정지, 클리어/패배 화면

3.2 룰
- 플레이어 시작 하트(Heart): 10
- 시작 골드: 300
- 타워 설치 비용(초기값):
  - 단일 저격형: 120
  - 광역 폭격형: 160
  - 감속 제어형: 140
- 하트 감소 규칙: 적 1기 누수(도착점 통과)마다 하트 1 감소
- 진행 구조: `한 판 = 한 Wave`
- 해금 규칙: Wave1 클리어 시 Wave2 해금, Wave2 클리어 시 Wave3 해금
- 웨이브 간 인터벌: 결과 화면 -> 상점(아이템 구매) -> 다음 Wave 진입
- 하트 초기화 규칙: 각 Wave 시작 시 Heart는 10으로 초기화
- 승리 조건: 해당 Wave 타이머 종료 또는 스폰 적 전부 처리 후 Heart > 0
- 패배 조건: Heart == 0

3.3 웨이브 구성(초기값)
- Wave1: 기본형만 등장 (베타 온보딩 웨이브)
- Wave2: 기본형 + 돌격형 혼합 등장
- Wave3: 기본형 + 돌격형 강화 물량 등장

3.4 플레이 시간 정책
- Wave1 최대 시간: 3분
- Wave2 최대 시간: 5분
- Wave3 최대 시간: 7분
- 총 최대 플레이 시간: 15분 (결과 화면 별도)

3.5 배속 정책
- 지원 배속: x1(기본), x2
- 배속 적용 대상: 적 이동, 공격 쿨타임, 투사체 진행, 스폰 주기, 이펙트 타이밍
- UI 정책: 현재 배속 상태를 HUD에 항상 표시

3.6 완료 기준 (Definition of Done)
- Wave1 단독 플레이 가능, 클리어 시 Wave2 해금 가능, Wave2 클리어 시 Wave3 해금 가능
- 타워 설치/공격/적 처치/골드 획득 정상 동작
- 누수 시 Heart 감소, 0 도달 시 패배 처리 정상
- Wave 종료 후 상점 진입 및 아이템 구매 후 다음 Wave 진입 정상 동작
- 각 Wave 시작 시 Heart 10 초기화 정상 동작
- 클리어/패배 화면 전환 정상
- 진행 불가/크래시급 버그 0건

3.7 테스트 체크리스트
- 설치 불가 타일에서 설치 거부 동작 확인
- 골드 부족 시 설치/업그레이드 거부 확인
- 웨이브 종료 및 승패 판정 정확성 확인
- 누수 1회당 Heart 1 감소 및 0 도달 즉시 패배 확인
- Wave1 클리어 후 Wave2 해금, Wave2 클리어 후 Wave3 해금 확인
- Wave 종료 후 상점 진입/아이템 구매/다음 Wave 진입 확인
- 각 Wave 시작 시 Heart가 10으로 초기화되는지 확인
- 앱 백그라운드 복귀 시 상태 무결성 확인
- 저사양 기준 30fps 이상 유지 확인

3.10 로비 및 스테이지 선택 시스템
- 유저는 앱 구동 시 "로비(Lobby) 화면"으로 진입하여 클리어한 스테이지(Wave) 목록을 확인하고 원하는 맵을 선택해 플레이할 수 있다.
- 진행 상황은 기기에 저장되어(로컬 스토리지), 앱을 재시작해도 해금 상태가 유지된다.

3.11 타워 조작 및 상호작용
- 타워의 상세한 설치 조건/건설 메뉴/업그레이드/판매 등 세부 플로우는 `TowerPlan.md`에서 분리 관리한다.

3.12 아이템 시스템 (베타)
- 아이템 상세 설계 문서는 `items.md`에서 관리한다.
- 재화 분리:
  - 골드: 전투 중 타워 설치/업그레이드
  - 다이아: 전투 보조 아이템 구매
- 베타 제한:
  - 아이템은 소모성 액티브 중심(2~3종)
  - 각 아이템은 판당 최대 1회 사용(초기값)
  - 배속(x2) 상태에서도 아이템 지속시간/효과는 실시간 기준으로 유지

3.9 그래픽/디자인 계획
- 그래픽 및 캐릭터 비주얼 상세는 `GraphicsPlan.md`에서 관리한다.
- 캐릭터 역할/스펙은 `Character.md`, 시각 디자인은 `GraphicsPlan.md`로 분리 관리한다.

4. 단계별 미션 (전체 로드맵)
1단계: 코어 루프(MVP) + 3웨이브 + x2 배속
2단계: 타워/적 상성 확장, 업그레이드 시스템 고도화
3단계: 난이도 모드(초보/중수/고수) 밸런싱
4단계: UI/UX 고도화 (메인 화면, 결과, 업그레이드, 피드백 연출)

5. 요청 사항
방금까지 대화하던 맥락을 이어받아, **[1단계: 프로젝트 구조 잡기]**부터 시작하고 싶어.
Cursor 터미널에 입력할 초기 설정 명령어와 기본 폴더 구조를 제안해줘.

## A. 난이도 수치표

### 목표
- 동일한 콘텐츠를 난이도별로 체감 차이 나게 제공하되, 규칙은 단순하게 유지한다.

### 범위
- 난이도는 `초보 / 중수 / 고수` 3단계
- 조정 대상: 다이아 보너스, 적 체력 배율, 난이도별 적 출현량, 적 출현 캐릭터 종류 수
- 고정 대상: 시작 골드(360), 플레이어 하트(각 Wave 시작 시 10), 적 이동속도 배율(1.00x), 처치 골드 배율(1.15x)

### 수치
| 항목 | 초보 | 중수(기본) | 고수 |
|---|---:|---:|---:|
| 시작 골드 | 360 | 360 | 360 |
| 플레이어 Heart | 10 | 10 | 10 |
| 적 체력 배율 | 0.85x | 1.00x | 1.20x |
| 적 이동속도 배율 | 1.00x | 1.00x | 1.00x |
| 처치 골드 배율 | 1.15x | 1.15x | 1.15x |
| 다이아 보너스(클리어) | +10% | 기준값 | -10% |
| 적 출현량 배율 | 0.90x | 1.00x | 1.20x |
| 적 출현 캐릭터 종류 수 | 1종 | 2종 | 전체 종 |

### 완료기준
- 난이도 선택 시 위 수치가 라운드 시작 시점에 정확히 적용된다.
- 난이도 변경은 인게임 중 불가, 로비에서만 가능하다.
- 난이도별 적 출현량 배율이 웨이브 총 스폰량에 반영된다.
- 난이도별 적 출현 캐릭터 종류 수가 표와 동일하게 적용된다.

### 테스트
- 각 난이도 시작 시 HUD 값(Heart/골드) 일치 확인
- 동일 웨이브에서 난이도별 적 생존시간 체감 차이 확인
- 동일 웨이브에서 난이도별 총 스폰량 증가(초보 < 중수 < 고수) 확인
- 동일 웨이브에서 난이도별 적 캐릭터 종류 수 증가(초보 < 중수 < 고수) 확인

상세 문서: `DifficultyPlan.md`

## B. 웨이브 스폰표 (Beta)

### 목표
- 짧고 명확한 웨이브 흐름으로 난이도 상승을 체감하게 한다.

### 범위
- Wave1~3, 각 웨이브 시간 3/5/7분
- Wave1은 적 1종, Wave2는 적 2종, Wave3는 현재 정의된 모든 적 캐릭터 등장
- 본 스폰표는 "Wave 스테이지 자체 설정"이며, 실제 출현 종류는 난이도 규칙(A)과 함께 적용

### 수치
| Wave | 시간 | 스폰 간격 | 구성 | 총 개체 수 |
|---|---:|---:|---|---:|
| Wave1 | 3분 | 2.2초 | 기본형 100% | 36 |
| Wave2 | 5분 | 1.8초 | 기본형 70% + 돌격형 30% | 90 |
| Wave3 | 7분 | 1.5초 | 모든 적 캐릭터 100% 순환 스폰(현재: 기본형 + 돌격형) | 168 |

난이도별 출현 종류 적용 규칙(우선순위):
- 초보: 각 Wave에서 1종만 출현
- 중수: 각 Wave에서 최대 2종 출현
- 고수: 각 Wave에서 해당 시점 정의된 전체 적 캐릭터 출현

### 완료기준
- 웨이브 시작 시 스폰 규칙이 표와 동일하게 적용된다.
- 웨이브 종료 후 다음 웨이브 전환이 2초 이내에 정상 동작한다.

### 테스트
- 스폰 카운트 로그로 총 개체 수 검증
- 비율 검증: 10회 샘플 실행 시 오차 ±5% 이내

상세 문서: `WavePlan.md`

## C. Wave별 맵 레이아웃 규격

### 목표
- 모바일 화면에서 가독성 높고 배치 판단이 쉬운 Wave별 전용 맵 3개를 만든다.

### 범위
- 9x16 그리드(세로형 기준), Wave별 고정 경로 1개
- Wave1/2/3 각각 서로 다른 맵 사용
- Wave3은 적 출현 입구 2개(듀얼 엔트리)
- 설치 가능 타일 10~12개, 주요 병목 2개 이상

### 수치
- Wave1 맵:
  - 시작점: `(0, 3)`, 도착점: `(8, 13)`, 경로 길이: 28타일
  - 설치 가능 타일: 10개
- Wave2 맵:
  - 시작점: `(0, 2)`, 도착점: `(8, 12)`, 경로 길이: 30타일
  - 설치 가능 타일: 11개
- Wave3 맵:
  - 시작점 A: `(0, 2)`, 시작점 B: `(0, 11)`, 도착점: `(8, 7)`
  - 경로 길이: A 24타일 / B 23타일 (중앙 합류형)
  - 설치 가능 타일: 12개
  - 병목 구간: 중앙 합류 직전 2구간

### 완료기준
- Wave1/2/3 맵이 서로 다르게 로드된다.
- 경로 이탈 없는 적 이동 가능
- 설치 가능 타일 외 위치 설치 불가
- Wave3 듀얼 엔트리(입구 2개) 동시 스폰 정상 동작
- 병목 구간에서 광역/감속 타워 효율 차이가 체감됨

### 테스트
- 50회 이동 시뮬레이션에서 각 Wave 경로 오류 0건
- Wave3 입구 A/B 동시 스폰 20회 반복 시 충돌/정지 오류 0건
- 임의 탭 테스트에서 설치 판정 오류 0건

상세 문서: `StagePlan.md`
