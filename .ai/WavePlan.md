# Wave Plan (Beta 4.0)

## 1) 목적
- `Wave 1~6`을 난이도(`easy/normal/hard`)별로 세분화해 밸런스 조정 기준표로 사용한다.
- 각 Wave마다 적 구성, 수량, HP, 속도, 골드/보석 보상을 명시한다.

## 2) 공통 기준
- 적 타입
  - `guard`: 기본 탱커형, 기준 HP 120, 처치 골드 10
  - `runner`: 고속 돌격형, 기준 HP 45, 처치 골드 12
- Wave 클리어 해금
  - Wave1 클리어 -> Wave2 해금 ... Wave5 클리어 -> Wave6 해금
- 보상 항목
  - `처치 골드`: Wave 내 적 처치로 누적
  - `클리어 골드`: Wave 성공 종료 시 추가
  - `클리어 보석`: Wave 성공 종료 시 지급

## 7) Time Objective (Wave 7, 시간제 웨이브)
- 시퀀스: `Wave 6` 클리어 후 자동 진입(로비에서는 선택하지 않음)
- 종료: `durationSec` 타이머 0 도달 시 즉시 `wave_clear` (단 `Heart == 0`이면 `game_over`)
- 진행: 타이머 동안 스폰/이동/누수/피해는 계속 진행
- HUD: 상단에 남은 시간 표시

### 보상 규칙(요청 반영)
- 골드: “마리당 기존 wave보다 적게”
  - time objective에서는 kill 골드를 0으로 두고, 대신 `wave_clear`에서 “마리 수 기반” 골드를 지급
  - guard/runner 기대 비율(40%/60%) 기준으로
    - guard: 1마리당 7골드 (floor(10 * 0.7) = 7)
    - runner: 1마리당 8골드 (floor(12 * 0.7) = 8)
  - 기대 평균: 약 `7.6 골드/마리`
- 보석: “10마리에 1개 정도”
  - `floor(totalEnemies / 10)`로 지급

## 10) 밸런스 튜닝 포인트
- `Wave4~6`에서 runner 비율이 급증하므로, 슬로우 타워/아이템 효율이 과도해지면 runner 수를 5~10% 감산한다.
- Hard에서 Heart 누수가 과하면 `Wave6 runner`를 162 -> 150으로 1차 완화한다.
- 골드 인플레이션 체크: `총 골드(예상)` 대비 타워 최종 업그레이드 타이밍을 기록해 상향/하향 조절한다.

## 11) 테스트 체크리스트
- 난이도별 `Wave1~6` 해금 순서 정상 동작
- Wave 종료 보상(골드/보석) 지급 정확성
- 적 수량/HP 적용값이 표와 일치하는지 로그로 검증
- Hard Wave6에서 최소 2회 이상 플레이 가능성(완전 불가능/완전 쉬움 배제) 확인

## 12) Beta 4.0 릴리즈 체크리스트(고정)
- 난이도별 진행도 분리(`easy/normal/hard`) 정상 동작
- 관리자 ON/OFF 시 다이아 값 원복 및 웨이브 오픈/잠금 정상 동작
- 상점 구매 쿨다운(0.25초) 및 인벤토리 상한(99) 정상 동작
- Wave7 time objective 타이머/UI/보상 규칙 정상 동작
- Wave1~15 맵 고유성 및 경로 이동 이상 없음 확인
