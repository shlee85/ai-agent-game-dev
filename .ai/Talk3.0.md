## Beta 3.0 실행 순서 (권장 고정안)

### Phase 0) 3D 기술 스택 고정 (1회)

```pseudo
GOAL: Expo/RN 환경에서 안정적으로 3D 렌더되는 최소 스택 확정

1. Choose rendering stack
   - expo-three + three + expo-gl (or equivalent Expo-compatible route)
2. Define asset format
   - glTF/glb as primary (mobile-friendly, tooling 호환성 높음)
3. Define folder convention
   - assets/3d/units/towers/
   - assets/3d/units/enemies/
   - assets/3d/vfx/
4. Define runtime contract
   - visualId <-> gameplayId(sniper, aoe, slow, runner, guard) 1:1 매핑
5. Exit criteria
   - blank scene + 1 test mesh + camera + light 정상 표시
```

### Phase 1) 유닛 3D MVP (프록시) (타워1 + 적1)

```pseudo
GOAL: "모델 로딩 + 배치 + 이동/타겟 동기화" 파이프라인 검증

1. Build proxy models
   - tower_sniper_proxy.glb
   - enemy_runner_proxy.glb
2. Replace 2D draw path only for selected units
   - 기존 전투 로직은 그대로 유지
3. Sync transforms
   - tile(row,col) -> world(x,z) 변환 함수 고정
   - enemy progress interpolation을 3D 위치로 반영
4. Add lightweight feedback
   - hit flash material tint
   - simple scale pulse for runner
5. Exit criteria
   - Wave 1에서 크래시 없이 플레이 가능
   - FPS/발열/메모리 문제가 없는지 1차 확인
```

### Phase 2) 유닛 5종 전체 3D화

```pseudo
GOAL: 타워3 + 적2를 전부 3D 유닛으로 치환

1. Add remaining unit proxies
   - tower_aoe_proxy, tower_slow_proxy, enemy_guard_proxy
2. Apply role-based silhouette rules
   - sniper: 길고 날카로운 실루엣
   - aoe: 둥근 reactor 실루엣
   - slow: 육각/결정형 실루엣
   - runner: 소형/빠른 실루엣
   - guard: 중형/둔중 실루엣
3. Validate readability
   - 탑다운 시점에서 즉시 구분 가능한지 확인
4. Exit criteria
   - 모든 유닛이 3D로 보이고 역할 식별 가능
```

### Phase 3) 발사/행동 동작 3D화

```pseudo
GOAL: 공격 타입/상태 피드백을 3D 연출로 전환

1. Attack VFX by type
   - single: 집중 임팩트 + 얇은 탄도/빔
   - slow: 냉기 링 + 감속 틴트
   - aoe: 확산 링 + 폭발 코어
2. Unit actions
   - tower recoil animation
   - enemy spawn pop-in
   - guard heavy bounce, runner trail
3. Keep gameplay timing unchanged
   - 쿨다운/데미지/속도 수치는 Beta2 기준 유지
4. Exit criteria
   - 시각만 3D로 바뀌고 게임 체감 밸런스는 동일
```

### Phase 4) 맵 3D화 (마지막)

```pseudo
GOAL: 타일/경로/환경을 3D화하되 가독성 유지

1. Build modular map kit
   - buildable tile, path tile, start, goal, obstacle
2. Camera policy
   - top-down 고정, 과한 perspective 금지
3. Lighting/material pass
   - 네온 강조 + 경로 가독성 최우선
4. UI overlap validation
   - HUD/팝업/타워 메뉴 가림 여부 점검
5. Exit criteria
   - 3D 맵에서도 경로/설치 가능 구분이 즉시 가능
```

- Unity에서 `Build And Run` 실행
- 앱이 바로 폰에 설치되어 테스트 가능

2. **APK/AAB 파일 테스트**
- 내부 테스트용 APK 생성 후 직접 설치
- 출시 직전에는 AAB 생성해서 Play Console 내부 테스트 트랙 업로드

3. **반복 테스트 루프 (권장)**
- Unity 수정 -> Build And Run -> 폰에서 확인 -> 수정 반복
