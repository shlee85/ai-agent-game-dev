# Tower Interaction & UI Plan

## 1) 문서 목적
- 본 문서는 타워 설치, 선택, 업그레이드, 판매 등 타워와 관련된 세부적인 UI/UX 상호작용 및 예외 처리 로직을 정의한다.
- 캐릭터 스펙은 `Character.md`에서, 전체 게임 룰은 `GamePlan.md`에서 분리하여 관리한다.

## 2) 타워 설치 (Build) 플로우

### 2.1 설치 조건 검사 (유효성)
유저가 그리드(타일)를 터치했을 때, 아래의 조건에 따라 처리가 달라진다.
- **설치 가능 타일 (빈 검은색 칸):**
  - 터치 시 해당 타일이 시각적으로 하이라이트(노란색 테두리 등) 처리된다.
  - 화면 하단에 **"건설 메뉴(Build Menu)"** UI가 팝업된다.
- **설치 불가 타일 (길, 시작점, 목표점):**
  - 터치 시 하이라이트 되지 않으며, "설치 불가" 시각적 피드백(빨간색 X 표시 0.5초 점멸) 또는 에러 효과음을 출력한다.
  - 기존에 열려있던 건설 메뉴가 있다면 닫힌다(취소).
- **이미 타워가 있는 타일:**
  - 터치 시 하이라이트 되며, 하단에 건설 메뉴가 아닌 **"타워 정보/업그레이드/판매 메뉴"** 가 팝업된다.

### 2.2 건설 메뉴 UI (하단 팝업)
빈 타일을 선택하여 건설 메뉴가 열렸을 때의 상세 동작:
- **메뉴 구성:** 현재 소지한 Gold로 설치할 수 있는 타워 리스트가 아이콘과 가격과 함께 표시된다.
- **비용 부족 (Gold 부족):**
  - 타워 아이콘이 비활성화(회색조, 자물쇠 표시 등)되며 터치해도 설치되지 않는다.
  - 터치 시 "골드 부족" 텍스트 알림이나 에러 피드백을 제공한다.
- **설치 진행:**
  - 비용이 충분한 타워를 터치하면 즉시 현재 소지한 Gold가 차감된다.
  - 선택했던 타일 위에 해당 타워의 그래픽(또는 임시 색상 아이콘)이 배치된다.
  - 설치 직후 하단 건설 메뉴는 자동으로 닫히고, 하이라이트가 해제된다.
- **취소 동작:**
  - 건설 메뉴 바깥의 맵 빈 공간, 다른 불가 타일, 혹은 메뉴 닫기 버튼을 누르면 메뉴가 사라지고 하이라이트가 해제된다.

## 3) 타워 관리 (업그레이드 & 판매) 플로우 [추후 개발]

### 3.1 관리 메뉴 UI
이미 설치된 타워를 터치했을 때 나타나는 상세 정보 메뉴:
- **표시 정보:** 타워 이름, 현재 레벨, 공격력, 공격 속도, 사거리
- **업그레이드 (Upgrade):**
  - 다음 레벨업에 필요한 Gold 표시.
  - 비용 부족 시 비활성화.
  - 터치 시 Gold 차감 후 타워 스탯(사거리, 데미지 등) 증가 및 외형/이펙트 변경.
- **판매 (Sell):**
  - 현재까지 투자된 총 Gold의 특정 비율(예: 70%)을 반환하며 타워를 맵에서 제거한다.
  - 터치 즉시 타워가 사라지고, 해당 타일은 다시 빈 칸(건설 가능) 상태로 돌아간다.

## 4) 타워 공격 판정 및 동작 (로직)
- **사거리 표시:**
  - 타워 건설 메뉴에서 타워 아이콘을 누르고 있거나(롱프레스), 맵에 이미 설치된 타워를 터치(선택)했을 때 해당 타워의 반경(사거리)이 맵 위에 시각적으로 옅은 원형으로 표시된다.
- **타겟팅 우선순위:**
  1. 목표점에 가장 가까운 적 (기본)
  2. 체력이 가장 높은 적 (저격형 등 특정 타워 옵션)
- **공격 쿨타임:** 각 타워 고유의 공격 속도에 따라 투사체를 발사하거나 즉발 피해를 입힌다. 배속(x2) 시 쿨타임도 2배 빠르게 감소한다.
