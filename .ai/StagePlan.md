# Stage Plan

## 1) 목적
- Wave별 맵 구성, 입구/출구, 병목, 설치 가능 타일 정책을 상세 정의한다.

## 2) 공통 규격
- 화면 기준: 세로형
- 그리드: 9x16
- 경로 타입: 고정 경로
- 설치 가능 타일: Wave별 10~12개

## 3) Wave별 맵 정의 (확정)

### Wave1 Map
- 시작점: `(0, 3)`
- 도착점: `(8, 13)`
- 경로 길이: 28타일
- 설치 가능 타일: 10개
- 목적: 온보딩(단순 경로, 단일 입구)

### Wave2 Map
- 시작점: `(0, 2)`
- 도착점: `(8, 12)`
- 경로 길이: 30타일
- 설치 가능 타일: 11개
- 목적: 중간 난이도(병목 활용 유도)

### Wave3 Map
- 시작점 A: `(0, 2)`
- 시작점 B: `(0, 11)`
- 도착점: `(8, 7)`
- 경로 길이: A 24타일 / B 23타일
- 설치 가능 타일: 12개
- 구조: 중앙 합류형 + 듀얼 엔트리
- 목적: 고난도(동시 처리 능력 검증)

## 4) 배치 원칙
- 병목 구간은 광역/감속 타워 효율이 드러나도록 배치
- 입구 근처 과도한 고정 배치로 난이도 붕괴가 발생하지 않게 제한
- 출구 인접 타일은 밸런스 이슈 시 설치 제한 가능

## 5) 테스트 체크리스트
- 각 Wave 경로 이탈 0건
- Wave3 듀얼 엔트리 동시 스폰 20회 안정성 확인
- 설치 가능/불가 타일 판정 정확성 확인
