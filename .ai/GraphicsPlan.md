# Graphics Plan

## 1) 목표
- 모바일에서 가독성이 높고, 타워/적/경로/위험도를 즉시 구분 가능한 비주얼을 만든다.
- 베타 단계에서는 "고퀄리티"보다 "명확한 식별성 + 성능 안정성"을 우선한다.

## 2) 아트 방향 (초기 확정안)
- 스타일: Stylized Sci-Fi + Clean UI
- 시점: 탑다운(Top-down)
- 톤: 채도 중간, 명도 대비 강하게
- 렌더링: 2D 스프라이트 기반 (성능 우선)

## 3) 컬러 시스템
- 아군(타워): 블루/시안 계열
- 적 유닛: 레드/오렌지 계열
- 경로: 중립 그레이 + 진행방향 하이라이트
- 위험 상태(하트 저하, 누수): 레드 경고 애니메이션
- 희귀/강화 상태: 퍼플/골드 보조색

## 4) 캐릭터별 디자인 컨셉

### 4.1 방어 캐릭터(타워)
- Aegis Lance (단일 저격형)
  - 실루엣: 길고 슬림한 캐논형
  - 키워드: 정밀, 관통, 고화력
  - VFX: 얇은 레이저 트레일, 단일 임팩트

- Ember Nova (광역 폭격형)
  - 실루엣: 넓은 베이스 + 원형 발사구
  - 키워드: 폭발, 범위, 다수처리
  - VFX: 원형 폭발 링, 잔광 파편

- Frost Ward (감속 제어형)
  - 실루엣: 육각 코어 + 냉각 핀
  - 키워드: 제어, 감속, 지속효과
  - VFX: 빙결 파동, 속도저하 오라

### 4.2 적 캐릭터
- Runner Drone (돌격형)
  - 실루엣: 작고 날카로운 삼각형 바디
  - 키워드: 빠름, 침투, 저체력
  - VFX: 잔상 트레일

- Guard Shell (기본/준탱커형)
  - 실루엣: 두꺼운 장갑형 바디
  - 키워드: 버팀, 전선, 고체력
  - VFX: 피격 시 장갑 스파크

## 5) UI/UX 비주얼 가이드
- HUD는 항상 상단 고정: Heart, 골드, 배속, 웨이브 정보
- 아이템 슬롯 3개는 하단 고정, 쿨다운/사용불가 상태 시 명도 하향
- 배속 상태(x1/x2)는 색상 + 텍스트 동시 표기

## 6) 애니메이션 원칙
- 핵심 전투 애니메이션은 Reanimated 기반으로 구현
- 상태 전환(승리/패배/경고)은 180~250ms 내 반응
- 과한 파티클보다 명확한 히트 피드백 우선

## 7) 제작 파이프라인 (AI 에셋 활용)
1. 콘셉트 키워드 정의
2. AI 이미지 생성(실루엣/색상안 3~5개)
3. 선택안 정제 및 스프라이트 시트화
4. 게임 내 임시 적용 후 식별성 테스트
5. 문제 자산 리터치 및 최종 반영

## 8) 품질 체크리스트
- 1초 내 타워/적 종류 식별 가능
- 색각 이상 환경에서도 아군/적 구분 가능
- 중저사양에서 프레임 저하 없이 VFX 표시
- 배속 x2에서도 이펙트 가독성 유지
